# Template Analysis — ZakaWise

## Comparison of GitHub Project Templates

| Template | Columns / Workflow | Automation Features | Agile Suitability |
|----------|------------------|---------------------|-------------------|
| Basic Kanban | To Do, In Progress, Done | No automation | Suitable for simple projects but lacks Agile automation |
| Automated Kanban | To Do, In Progress, Done | Auto-move issues based on status | Highly suitable for Agile workflows and sprint tracking |
| Bug Triage | New, Needs Review, Closed | Auto-labeling and prioritization | Best for bug handling, not full system development |
| Team Planning | Backlog, Ready, In Progress, Done | Supports planning and prioritization | Good for planning but less flexible than Kanban |

---

## Selected Template: Automated Kanban

### Justification

The **Automated Kanban template** was selected because it aligns best with the Agile development approach used in the ZakaWise system.

Firstly, it supports **automation**, allowing issues (user stories and tasks) to automatically move between columns such as "To Do", "In Progress", and "Done" based on their status. This reduces manual work and improves workflow efficiency.

Secondly, it fits well with **Sprint Planning from Assignment 6**, where user stories are implemented incrementally. The automation ensures that task progress is clearly visible without constant manual updates.

Additionally, the template supports **continuous delivery**, which is a key Agile principle. As tasks are completed, they are automatically moved, allowing the developer to focus more on development rather than project tracking.

Compared to other templates:
- Basic Kanban lacks automation
- Bug Triage is too focused on bugs only
- Team Planning is more suitable for large teams

Therefore, Automated Kanban is the most suitable for managing the ZakaWise Agile workflow.
